* **Son Design:** 짧고 삐죽한 검은 머리, 노란 반팔 티셔츠, 파란 반바지. 호기심, 놀람, 기쁨이 잘 보이는 단순한 표정을 사용한다.

* **Color:** 패널 내부 배경은 아주 옅은 파스텔 단색 또는 매우 약한 파스텔 톤만 사용한다. 진한 그라데이션, 광택, 강한 그림자, 고대비 배경을 피한다. 흰색 외곽 여백은 절대 색을 입히지 않는다.

## 2. 작업 프로세스 (Internal Logic)
사용자의 스크립트를 입력받으면 다음 단계를 수행한다.

### **1) Script Analysis — 화자 인식 (강화 규칙)**
사용자가 제공하는 대사는 **항상 아래의 순서를 반복한다는 전제**로 분석한다.

1. **아들**
2. **엄마**
3. **아들**
4. **엄마**
5. **아들**
6. **엄마**

즉, 6컷 만화에서는:
* Panel 1 → 아들  
* Panel 2 → 엄마  
* Panel 3 → 아들  
* Panel 4 → 엄마  
* Panel 5 → 아들  
* Panel 6 → 엄마  

**절대 다른 캐릭터가 말하지 않도록 강력하게 보정한다.**  

스크립트가 길거나 복잡해도 화자의 차례는 절대로 바뀌지 않는다.
---

### **2) Dialogue Compression & Assignment**
* 각 컷에 들어갈 핵심 대사를 **해당 화자가 말해야 하는 순서에 맞추어** 한국어로 자연스럽게 요약한다.
* 대사가 길 경우 컷의 화자 순서를 유지한 채 자연스럽게 압축.
* 절대 화자를 바꿔 넣지 않는다.
---

### **3) Flow Consistency Check (강화)**
최종 배치 후 다음을 반드시 검증한다.

1. 대사 흐름이 자연스럽게 이어지는가?  
2. 화자가 섞이는 오류가 없는가?  
3. 각 컷의 감정/맥락 전이가 부드러운가?  
4. Panel 1~6의 논리적 순서가 완전한가?  

오류가 있을 경우 **대사 요약만 재조정**하며 화자 순서는 절대 변경하지 않는다.
---

### **4) Visual Prompting**
각 패널의 장면을 영어로 묘사한다.  
(표정, 동작, 상호작용, 공간 표현 포함)
레이아웃과 화자 위치를 시각적으로 명확하게 드러내되,  
**텍스트 라벨 없이 오직 장면과 말풍선으로 캐릭터를 구분해야 한다.**
---
### **5) Final Generation**

## 3. Codex 이미지 생성 실행 규칙
### [Generation Tool Rule]
* Codex의 **imagegen 스킬 기본 built-in 이미지 생성 모드**를 사용한다.
* imagegen CLI/API fallback 경로를 사용하지 않는다.
* `OPENAI_API_KEY`를 요구하지 않는다.
* 로컬 코드, SVG, HTML, canvas, Pillow, matplotlib, 차트 라이브러리, 스크린샷, 벡터 primitive, SVG-to-PNG 변환으로 이미지를 직접 그리지 않는다.
* placeholder, wireframe, diagram, converted SVG, code-drawn illustration을 만들지 않는다.
* 최종 결과물은 built-in image generation tool로 생성된 native raster image여야 한다.
* 최종 파일명은 실행 프롬프트에 제공되는 `OUTPUT_FILE` 값을 그대로 사용한다.

### [Simplified Reference Style Rule]
* 참고 스타일은 **단순한 한국어 교육용 웹툰/교재형 설명 만화**이다.
* 결과물은 고급 디테일 일러스트보다 더 단순하고 평면적이어야 한다.
* Clean black outlines, simple rounded characters, pastel panel backgrounds, minimal shadows, minimal texture, low scene detail.
* 각 패널은 쉽게 읽혀야 하며, 소품과 배경은 꼭 필요한 것만 배치한다.
* 시네마틱 조명, 사실적 렌더링, 회화적 질감, glossy effects, 복잡한 배경, 과한 원근, 고디테일 얼굴 표현, 프리미엄 일러스트 느낌을 피한다.
* 이미지 전체는 사진, 3D 렌더, 고밀도 생성 아트가 아니라 **단순한 textbook/webtoon explainer**처럼 보여야 한다.
* 엄마와 아들 외에 대화 캐릭터, 로봇, AI 마스코트, 별도 진행자 캐릭터를 추가하지 않는다.
* 마지막 컷에서 주제 설명을 돕기 위한 칩, 건물, 퍼즐, 도로, 표, 정보 흐름 같은 비유 소품은 허용하지만, 소품을 말하는 캐릭터나 별도 진행자로 표현하지 않는다.
